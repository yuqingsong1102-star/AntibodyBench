# Antibody Dataset Index Files

Use these files in order. They represent different stages of data readiness.

## 1) `dataset_index.csv` (base index)

Primary working index used by scripts.

Current key columns:

- `sample_id`
- `pdb_id`
- `reference_structure_path` (often antigen-focused/cropped path)
- `antigen_chain`
- `antibody_chain`
- `cdr_h3_start`
- `cdr_h3_end`
- `reference_complex_path` (full complex path, preferred for antibody numbering)
- `reference_complex_status`
- `reference_complex_note`

## 2) `dataset_index_h3_annotated.csv` (annotation audit output)

Generated by:

- `python scripts/data_prep/fill_cdr_h3_from_anarci.py ...`

Adds annotation status columns:

- `cdr_h3_status` (`ok` / `skip` / `fail`)
- `cdr_h3_note` (reason details)

Use this file to inspect failures and data quality.

## 3) `dataset_index_ready.csv` (filtered ready subset)

Subset for actual evaluation runs.
Usually includes only rows with:

- `cdr_h3_status == ok`
- non-empty `cdr_h3_start` and `cdr_h3_end`

Use this file with pipeline runs when you want stable metric output.

Example:

```bash
bash scripts/core/run.sh --model RFantibody --split-csv inputs/antibody_datasets/dataset_index_ready.csv
```

## Recommended workflow

1. Generate/refresh base index: `generate_dataset_index.py`
2. Fill full-complex paths: `fetch_reference_complexes.py`
3. Annotate H3: `fill_cdr_h3_from_anarci.py`
4. Filter ready subset: produce/update `dataset_index_ready.csv`
5. Run model evaluation using ready subset
